# 繰り返しは RFC 5545 RRULE 形式で保存する

Notification の繰り返し（週次・隔週・毎月第 N 曜＋単発）を、自前のルールフィールド群ではなく **RFC 5545 の RRULE 文字列**で保存する。繰り返し計算の自作（月境界・第 N 曜・例外・間隔）は典型的な沼であり、標準フォーマットに乗ることで表現力と将来拡張（隔月・毎年等）を安く確保する。

評価は `rrule.js` の `.after(now)` で「次の開催回」を求めることを第一候補とする。ただし Cloudflare Workers(workerd) 上での動作は**テストで検証**し、もし非互換なら、サポート対象サブセット（週次・隔週・毎月第 N 曜・単発）のみを評価する自前エバリュエータをフォールバックとする（保存形式は RRULE 文字列のまま維持し、消費側 API は変えない）。

トレードオフ: `rrule.js` への依存（ロックイン）と Workers 互換性の検証コスト。日付評価は JST 固定（[ADR 0003](./0003-generality-boundaries.md)）で、tzid は使わない。

> **補足（UI 範囲）**: 管理 UI が生成する繰り返しはビルダのプリセット（週次／隔週／毎月第N曜＋単発）に意図的に限定し、生 RRULE は提示しない（[ADR 0007](./0007-admin-ui-redesign.md)）。保存形式は RRULE 文字列のままで本 ADR の表現力（隔月・毎年等）は維持するが、プリセット外パターンの作成は将来のビルダ拡張に委ねる。

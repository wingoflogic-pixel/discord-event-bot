# 汎用化の境界（明示的な非目標）

「汎用化」は**構造の汎用化**（複数 Event / Notification / Segment / スケジュール）に限定し、次の 3 つは意図的に固定する。

1. **単一テナント** — 1 デプロイ = 1 サーバー。マルチテナント（`guild_id` 分離・Bot 招待時オンボーディング・認可境界）は次弾に分離。
2. **言語 = 日本語固定** — 応答ステータス（参加 / 不参加 / 未定）とメッセージ文言はハードコード。i18n は行わない。
3. **時刻 = JST 固定** — タイムゾーン設定を持たず、`getJSTNow()` 基準で日付・RRULE を評価する。

理由: 自己ホストの単一テナントであり、当面の対象が日本語コミュニティという前提では、i18n・TZ・マルチテナントは労多く益少ない。まず構造の汎用化（複数イベント・複数対象・繰り返し・番号割り当て）に集中する。将来の読者が「なぜ"汎用"なのに JST / 日本語固定なのか」を理解できるよう、この境界を明示しておく。

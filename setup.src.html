用意するもの

登録は Discord・Cloudflare・GitHub の3つ・すべて無料 ▶
  • Discord アカウントと、Bot を入れたいサーバーの管理権限
  • Cloudflare アカウント（無料。クレジットカード不要）
  • GitHub アカウント（無料。リポジトリの複製（fork）と、更新時の「Sync fork」に使います）

このアシスタントでは、設定に必要な「合言葉」を順に集めていきます。各欄に貼り付けると、次の手順で自動的に使い回します（招待リンクの生成・コピー用の値など）。

1

Discord でアプリ（Bot）を作る

3つの合言葉を取得し、Bot をサーバーに招待 ▶

Discord の開発者ページで Bot を作り、設定に必要な3つの合言葉（アプリID・公開キー・Bot のトークン）を集めます。まず下のボタンでページを開いてください（画面が日本語の場合の表記で説明します）。

1アプリを新規作成する

画面の右上▸青い「新しいアプリケーション」ボタンを押します。出てきた「アプリを新規作成する」の窓で 名前（例: イベント出欠Bot）を入れ、規約のチェックを入れて 「作成」 を押します。

2「アプリID」と「公開キー」をコピーする

左メニュー▸概要▸一般情報を開き、少し下にスクロールします。「アプリID」と「公開キー」それぞれの「コピー」ボタンを押し、下の入力欄に貼り付けてください。

数字だけのIDを貼り付けてください。

3Bot のトークンを取得し、メンバー取得を ON にする

左メニュー▸Botを開き、「トークン」の「トークンをリセット」（青ボタン）を押すと、新しいトークンが表示されます。コピーして下の欄に貼り付けてください（トークンは表示が一度きりです）。

続けて、同じ Bot ページを下にスクロールし、「Server Members Intent」のスイッチを ON（右側・青） にします（メンバー一覧の取得に必要）。

4「インストールリンク」を「なし」にする

左メニュー▸概要▸インストールを開き、「インストールリンク」のドロップダウンを「なし」に変更します（自分専用の Bot にするため）。これをやらないと、あとで Private application cannot have a default authorization link というエラーになります。

💡 Bot をサーバーに招待：アプリIDを入れると、必要な権限を組み込んだ招待リンクを自動で作ります。
招待リンク （アプリIDを入力すると表示されます）
付与される権限：チャンネルの閲覧 ／ メッセージの送信 ／ ロール・@everyone へのメンション（募集メッセージの通知に使用）。
2

管理パスワード（ADMIN_TOKEN）を作る

管理画面に入るための合言葉・自動生成できます ▶

この Bot の管理画面に入るためのパスワードです。推測されにくい長い文字列が必要なので、下のボタンで自動生成します（自分で考えなくてOK）。

⚠️ このパスワードはあとで「デプロイ時の入力」と「管理画面ログイン」で使います。生成したらコピーして安全な場所に控えてください。
3

GitHub で複製して Cloudflare に接続する

fork → Cloudflare に接続 → 4つの合言葉を設定 ▶

更新を「ボタン1つ」で受け取れるように、まず公開リポジトリを自分の GitHub に複製（fork）し、それを Cloudflare に接続します。ターミナルは使いません。（GitHub・Cloudflare の画面は英語表記です。）

1リポジトリを fork する

上の「リポジトリを fork する」を開き、GitHub にログイン。「Owner」▸あなたのアカウントを選び、右下の緑「Create fork」を押します。あなたの GitHub にコピー（fork）ができます。

2Cloudflare に接続する

上の「Cloudflare ダッシュボードを開く」を押すと、「Workers & Pages」の画面が開きます。右上の青「Create application」を押します。

次の画面で「Continue with GitHub」を押します（初回は GitHub 連携を許可）。

リポジトリ一覧から、さっき fork した「discord-event-bot」の行をクリックして選びます。選ぶまで右下の「Next」は灰色（押せません）。選ぶと押せるようになるので「Next」を押します。

3Deploy command を「npm run deploy」に変える

次の「Set up your application」画面で、「Deploy command」の欄を、最初に入っている npx wrangler deploy から下の値に書き換えます（ここを変えないと、更新時にデータベースの更新が走りません）。

Deploy commandnpm run deploy

4デプロイする

同じ画面の右下、青い「Deploy」を押します。データベース（D1）は自動で作られ、表も自動で用意されます。

54つの合言葉（シークレット）を設定する

作成された Worker の Settings▸Variables and secrets を開き、「+ Add」から下の4つを1つずつ追加します。手順1の3つ＋手順2のパスワードです。

DISCORD_PUBLIC_KEY（手順1で入力）
DISCORD_APPLICATION_ID（手順1で入力）
DISCORD_BOT_TOKEN（手順1で入力）
ADMIN_TOKEN（手順2で生成）

6Bot の URL を控える

Worker の Overview（概要）画面の上に出る https://◯◯◯.workers.dev が、あなたの Bot の URL です。下の欄に貼り付けてください。

https:// で始まる URL を貼り付けてください。
💡 4つの合言葉は後から追加してもOK（追加すると即反映されます）。合言葉が未設定だと Bot は反応しません。
4

管理画面を開いてコマンドを登録する

スラッシュコマンドをボタンで登録 ▶
  1. 下のボタンで管理画面を開き、ADMIN_TOKEN（手順2のパスワード）でログイン。
  2. 左メニュー 「⚙️ セットアップ」 を開き、シークレットがすべて「設定済み」か確認。
  3. 「コマンドを登録」ボタンを押す。下記の 3 つが Discord で使えるようになります:
    • /recruit（管理者用）— 募集メッセージをそのチャンネルに今すぐ送信
    • /help（誰でも）— コマンド一覧と管理画面 URL を表示
    • /manage（管理者用）— 管理画面の URL を表示
ログイン用 ADMIN_TOKEN（手順2で生成）
先に手順3で Bot の URL を入力すると、ここから直接開けます。
5

Discord に接続先（URL）を貼る

Interaction Endpoint URL の設定 ▶

Discord からのボタン操作やコマンドを Bot が受け取れるように、接続先の URL を Discord に登録します。

  1. 下の URL の「コピー」を押す。
  2. 「Discord Developer Portal を開く」→ 左メニュー 概要▸一般情報を開いて下にスクロール → 「インタラクション・エンドポイントURL」の欄に貼り付け。
  3. ページ下部に出る「変更を保存する」を押す（Discord が自動で接続確認 → 緑のチェックで成功）。
エンドポイントURL（手順3で URL を入力）
6

中身を作る（区分 → 通知 → メンバー）

管理画面で実際のイベント設定 ▶

ここからは管理画面の左メニューで設定します。順番は 「メンバー区分」→「通知」 です。

1メンバー区分をつくる

左メニュー▸👥 メンバー区分を開き、「新規区分」で対象グループを作成（例: キャスト、スタッフ）。@メンションする Discord ロールは任意で紐付け。「名前」を入れて「保存」。メンバーは各区分の「メンバー」ボタンから一覧で選ぶだけです。

2通知（募集）をつくる

左メニュー▸🔔 通知を開き、対象区分（メンバー区分）・投稿チャンネル・種別（繰り返し/単発）・開始時刻・曜日などを設定して保存。設定どおりに募集とリマインドが自動で送られます。

メンバーは募集メッセージのボタンに回答すると自動で登録され、区分にも自動所属します。

📖 管理画面の各メニューは、画面のラベルに沿って進められます。
🔑

ADMIN_TOKEN を忘れたとき

あとから確認はできません。新しく作り直せばOK（データは消えません） ▶

ADMIN_TOKEN（管理画面のパスワード）は、安全のため「あとから中身を見て確認する」ことができません。でも心配いりません。忘れてしまったら、新しいパスワードに作り直すだけで、また管理画面に入れます。

😌 データは消えません：作り直しても、登録済みのメンバー・回答・通知などはそのまま残ります。変わるのは「ログインの合言葉」だけです。

1新しいパスワードを用意する

このページの上の方にある「手順2：管理パスワード（ADMIN_TOKEN）を作る」を開き、「🎲 パスワードを生成する」を押すと、新しい合言葉を作れます。表示された文字列をコピーし、安全な場所（パスワード管理アプリなど）に控えてください。

💡 自分で考えた長い文字列でも構いませんが、推測されにくいよう英数字で30文字以上を目安にしてください。

2Cloudflare で古いパスワードを差し替える

手順3で使った Cloudflare の画面を開き、あなたの Worker▸Settings▸Variables and secrets を開きます。一覧の中の ADMIN_TOKEN の行で、次のどちらかを行います。

  1. 行の右の「Edit」（編集）を押し、①で作った新しい値に書き換えて保存する。
  2. 「Edit」が見当たらないときは、ADMIN_TOKEN を「Delete」（削除）してから、まったく同じ名前でもう一度「+ Add」を押し、新しい値を入れて保存する。
設定する名前（コピーして貼り付け）ADMIN_TOKEN
⏱️ 保存すると数秒で反映されます。名前は ADMIN_TOKEN と1文字も違わないようにしてください（前後に空白を入れない・大文字小文字もそのまま）。値が Value encrypted と表示されればOKです。

3新しいパスワードでログインする

管理画面をもう一度開き、①で作った新しいパスワードを入力してログインします。これで完了です。

🔒 安全のヒント：古いパスワードは差し替えと同時に使えなくなります。「誰かに知られたかも」というときも、この手順で作り直せば安心です。
💡 （パソコンに詳しい方向け）すぐ確認したいとき：管理画面にまだログインしたままのブラウザがあれば、そこに今のパスワードが残っています。そのブラウザで F12 →「Application（アプリケーション）」→「Local Storage」→ あなたの Bot の URL を選ぶと、eventbot_admin_token の値が現在の ADMIN_TOKEN です。
🆘

うまくいかないときは

よくある症状と確認すること ▶
  • スラッシュコマンド（/recruit /help /manage）が Discord に出てこない
    手順4「コマンドを登録」を実行したか／手順5 の Interaction Endpoint URL を保存したか。
  • Discord で「インタラクション・エンドポイントURL」の保存時に「The specified interactions endpoint url could not be verified」エラー
    Cloudflare 側の DISCORD_PUBLIC_KEY が未設定 or 誤りです。手順1② の公開キーを Cloudflare の Settings → Variables and secrets で正しく設定し直してください（保存すれば数秒で反映）。その後、Discord 側でもう一度「変更を保存する」を押します。それでも直らないときだけ、Worker を再デプロイ（Deployments タブから Retry）。
  • Discord で「Private application cannot have a default authorization link」エラー（PUBLIC BOT を OFF にできない）
    Discord Developer Portal → 左メニュー 概要▸インストール → 「インストールリンク」 を 「なし」 に変更（手順1④）。
  • 「サーバーが取得できません」と出る
    Bot をサーバーに招待したか（手順1）／DISCORD_BOT_TOKEN が正しいか。
  • メンバー一覧（ピッカー）が空
    Discord 側で Server Members Intent を ON にしたか（手順1）。
  • 募集メッセージが投稿されない
    Bot にそのチャンネルへの送信権限があるか／通知の投稿チャンネル設定。
  • @ロールのメンションが飛ばない
    招待時に Mention Everyone 権限を付けたか／対象ロールが「メンション可能」か。
  • 管理画面にログインできない
    ADMIN_TOKEN（手順2）が一致しているか／管理画面の「⚙️ セットアップ」で「設定済み」か。
  • Cloudflare で初回ビルドが走らない（Deployments タブに新しいエントリが出ない）
    fork したリポジトリで何か変更を 1 回 push してください（GitHub Web 上の README 編集など、小さな変更でも OK）。Cloudflare Workers Builds は接続直後にビルドを自動起動せず、次の push を待つ仕様です。
  • 更新後にデータが消えたように見える
    同じ Worker に更新していればデータは保持されます。Cloudflare の Worker → Settings → Bindings で DB（D1）の紐付けを確認。同じアカウント内に同名 D1 があれば再利用されるので、Worker を作り直しても多くの場合はデータが維持されます（保証ではないため、重要なデータは管理画面からエクスポートしてバックアップしてください）。
🔄

アップデートのしかた

新バージョンが出たら GitHub の fork で「Sync fork」を押すだけ ▶

新しいバージョンが出たら、自分の GitHub の fork ページ（手順3 で作ったコピー）を開いて 「Sync fork」 ボタンを押すだけです。

  1. Cloudflare が変更を検知して自動で再デプロイします。
  2. データベースも自動で最新化されます（手作業のマイグレーションは不要）。
💡 「DB変更あり」と書かれたリリースでも、やることは同じ 「Sync fork」 だけ。必要なデータベースの更新も自動で適用されます。ターミナル操作は不要です。
⚠️ この自動更新は、手順3 ③の Deploy command が npm run deploy のままであることに依存します（素の wrangler deploy に変えると、データベースの自動更新が止まります）。同じ Worker に更新する限り、これまでのデータ（メンバー・回答など）は保持されます。
🎉 おつかれさまでした！ セットアップは完了です。Bot が募集・リマインドを自動で行います。 共用パソコンの場合は、下の「入力をすべて消去」を押して合言葉を消しておきましょう。
